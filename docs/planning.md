## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 웰마취통증의학과의원 공식 웹사이트 |

## 2. 사이트맵

```
웰마취통증의학과의원
├── 공개 페이지
│   ├── / (홈)
│   ├── /about (병원 소개)
│   ├── /treatment (진료안내)
│   │   └── /treatment/[slug] (진료과목 상세)
│   ├── /doctors (의료진 소개)
│   ├── /notice (공지사항)
│   │   └── /notice/[id] (공지사항 상세)
│   └── /contact (오시는 길 & 연락처)
│
└── 관리자 페이지 (URL 직접 접근 전용 — 공개 네비게이션 노출 없음)
    ├── /admin/login (로그인)
    ├── /admin (대시보드)
    └── /admin/notice (공지사항 관리)
        ├── /admin/notice/new
        └── /admin/notice/[id]/edit
```

## 3. 공개 페이지 상세

### 3.1 홈 (`/`)

**목적:** 방문자의 첫인상, 핵심 정보 빠른 접근

| 섹션 | 내용 |
|------|------|
| Hero | 병원명, 슬로건, 대표 이미지 |
| 네이버 리뷰 슬라이더 | 실제 네이버 환자 리뷰 카드를 가로 스크롤로 표시 — 자동 슬라이드(Auto-scroll) + 드래그/터치 스와이프 지원, 진입 시 fade-in 애니메이션 |
| 진료과목 요약 | 주요 진료과목 3–4개 카드 |
| 병원 소개 요약 | 한 줄 소개 + 소개 페이지 링크 |
| 의료진 소개 요약 | 의료진 프로필 카드 1–2개 |

### 3.2 병원 소개 (`/about`)

**목적:** 병원 철학 및 신뢰 구축

| 섹션 | 내용 |
|------|------|
| 인사말 | 원장 인사말 (텍스트 + 사진) |
| 병원 특장점 | 차별화 포인트 3–5가지 |
| 시설 소개 | 시설 사진 갤러리 |
| 연혁 | 병원 개원 이후 주요 이력 (선택) |

### 3.3 진료안내 (`/treatment`)

**목적:** 어떤 질환을 치료하는지 명확히 안내

| 섹션 | 내용 |
|------|------|
| 진료과목 목록 | 과목별 카드 (아이콘, 이름, 한 줄 설명) |
| 진료시간 | 요일별 진료시간 표 |
| 진료과목 상세 | 질환 설명, 증상, 치료 방법 |

**진료과목 예시 (마취통증의학과):**
- 허리/목 통증
- 신경치료 (신경차단술)
- 관절 통증 (무릎, 어깨 등)
- 두통 / 어지럼증
- 대상포진 후 신경통

### 3.4 의료진 소개 (`/doctors`)

**목적:** 전문의 신뢰도 제공

| 필드 | 내용 |
|------|------|
| 사진 | 의료진 프로필 사진 |
| 이름 / 직함 | 원장, 과장 등 |
| 전문 분야 | 세부 전문 분야 태그 |
| 약력 | 학력, 경력, 자격증 등 |

### 3.5 공지사항 (`/notice`)

**목적:** 휴진 안내, 이벤트, 병원 소식 공유

| 기능 | 설명 |
|------|------|
| 목록 | 제목, 날짜, 휴진여부 배지 |
| 상세 | 제목, 본문, 날짜 |
| 페이지네이션 | 10건씩 |
| 휴진 표시 | 휴진 공지는 별도 뱃지 표시 |

### 3.6 오시는 길 (`/contact`)

**목적:** 방문 접근성 극대화

| 섹션 | 내용 |
|------|------|
| 주소 | 도로명 주소 |
| 전화번호 | 클릭 시 전화 연결 (`tel:`) |
| 지도 | 카카오맵 또는 네이버 지도 임베드 |
| 교통 안내 | 대중교통, 주차 정보 |
| 진료시간 요약 | 간단한 진료시간 표 |

## 4. 관리자 페이지 상세

**접근 제어:** URL 직접 접근 전용 (`/admin/login`) — 공개 UI에 로그인 버튼 없음. `middleware.ts`에서 `/admin/*` 요청 시 세션 없으면 `/admin/login`으로 리다이렉트.

### 4.1 대시보드 (`/admin`)

| 섹션 | 내용 |
|------|------|
| 공지사항 최근 5건 | 빠른 링크 |
| 빠른 메뉴 | 공지사항 관리 바로가기 |

### 4.2 공지사항 관리 (`/admin/notice`)

| 기능 | 설명 |
|------|------|
| 목록 조회 | 제목, 날짜, 휴진여부, 수정/삭제 버튼 |
| 작성 | 제목, 본문(마크다운), 날짜, 휴진여부 토글 |
| 수정 | 기존 데이터 불러와 편집 |
| 삭제 | 확인 다이얼로그 후 삭제 |

> 진료안내, 의료진, 병원 소개 데이터는 코드에 하드코딩으로 관리 — DB/관리자 불필요

## 5. 데이터 모델

### 현재 구현됨
```prisma
model Notice {
  id         Int      @id @default(autoincrement())
  title      String
  content    String
  date       DateTime
  isHoliday  Boolean  @default(false)
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt
}
```

> 추가 모델 없음. 공지사항 외 모든 콘텐츠는 `src/shared/config/` 또는 각 슬라이스 내 TypeScript 파일에 하드코딩.

## 6. UI/UX 방향

| 항목 | 방향 |
|------|------|
| 색상 | 신뢰·전문성을 주는 블루 계열 (메인) + 화이트 (배경) |

| 성능 목표 | Lighthouse 90점 이상 (모든 항목) |
